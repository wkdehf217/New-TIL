# Docker

> Docker 복습이 목표

### Docker가 쓰이는 이유
  - 일관된 개발 환경이 필요할 때 
  - Docker 이미지로 컨테이너 빌드 후 필요한 모든 구성 요소가 포함되어 있어 별도의 설치 과정 없이 바로 실행할 수 있음
  - CI/CD 파이프라인을 구축할 때, 코드를 변경할 때마다 자동으로 빌드, 테스트, 배포할 수 있도록 설정 가능
    <br/><br/>\* 말고도 여러가지 적지 않은 이유가 있음

- 💡 키워드 정리
```
- 이미지        : 애플리케이션과 모든 실행에 필요한 파일을 포함한 읽기 전용 템플릿
- 컨테이너      : 이미지를 실행하여 동작하는 애플리케이션 인스턴스
- Dockerfile    : 이미지를 생성하기 위한 명령어가 담긴 스크립트 파일
- Docker Hub    : 이미지를 저장하고 공유하는 중앙 저장소
- 볼륨          : 컨테이너 데이터를 지속적으로 저장하는 메커니즘
- 네트워크      : 컨테이너 간의 통신을 관리하는 방식
```

### Docker Image?
  - 쉽게 말해서 Docker Image는 도커를 무한으로 찍어낼 수 있는 복사 템플릿 같은 것 
  - Docker 는 Docker Image를 사용해서 빌드를 함
  - Docker Image는 소스 코드, 라이브러리, 종속성, 도구 및 응용 프로그램을 실행하는데 필요한 기타 파일을 포함하는 불변(변경 불가) 파일
 
### Docker Compose
  - Docker Compose는 다중 컨테이너 Docker 애플리케이션을 정의하고 실행하기 위한 도구
  - docker-compose.yml 파일 하나로 애플리케이션의 서비스, 네트워크, 볼륨 등을 정의할 수 있음
  - 중요 기능은 컨테이너 내부에서 실행되는 컴퓨팅 환경의 표준화

### 도커 이미지와 컨테이너의 차이점은? Docker Images vs Container

- 1. 개념
  - Docker에서 Image와 Container는 서로 밀접한 관계를 가진다.

    - Image는 실행 환경이 정의된 정적인 템플릿
    - Container는 Image를 실행한 동적인 인스턴스

  - 컨테이너는 이미지를 기반으로 생성되기 때문에 이미지 없이 컨테이너는 존재할 수 없지만, 이미지는 단독으로 존재할 수 있다.

- 2. 동작 흐름
  ```
  Dockerfile → Image → Container
  Dockerfile   : 이미지 생성을 위한 설정 파일
  Image        : 실행 환경이 저장된 결과물
  Container    : 이미지를 실행한 상태
  ```
- 3. Image

  - Image는 애플리케이션 실행에 필요한 환경을 포함한 패키지이다
  
    - OS, 라이브러리, 애플리케이션 코드 포함
    - 변경되지 않는 정적인 구조(Read-only)
    - 컨테이너 실행의 기반이 되는 템플릿
  
  - 즉, Image는 실행 환경을 그대로 저장해둔 결과물이다

- 4. Container

  - Container는 이미지를 실행한 결과이다
  
    - 실행 시 Writable Layer가 추가됨
    - 각 컨테이너는 독립적인 상태와 데이터를 가짐
    - 동일한 이미지로 여러 개 생성 가능
  
  - 컨테이너는 격리된 환경에서 실행되는 프로세스라고 볼 수 있다

- 5. Image vs Container 정리
  ```
  구분	          Image	              Container
  성격	          정적	              동적
  역할	          실행 환경 정의	  실제 실행
  변경 가능 여부	  불가능	          가능
  관계	          컨테이너의 기반	  이미지의 실행 결과
  ```
- 6. 핵심 정리
  
  - 컨테이너는 이미지를 실행한 인스턴스이다
  - 이미지는 변하지 않지만, 컨테이너는 실행되면서 상태가 변한다
  - 하나의 이미지로 여러 개의 컨테이너를 생성할 수 있다
  - 컨테이너는 격리된 환경에서 애플리케이션을 실행한다

    <br/>
- 7. 추가 이해
  
  - 컨테이너가 생성된다는 것은<br/>
  단순히 프로그램을 실행하는 것이 아니라,<br/>
  특정 실행 환경(OS, 라이브러리 등)을 포함한 상태로 프로세스를 격리하여 실행하는 것이다
