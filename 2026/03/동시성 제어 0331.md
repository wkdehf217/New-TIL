# 동시성 제어


### 트랜잭션
- 트랜잭션: DBMS에서 데이터를 다루는 논리적인 작업의 단위
  
- DB에서 데이터를 다룰 때 장애가 일어난 경우 데이터를 복구하는 작업의 단위가 된다
  
- DB에서 여러 작업이 동시에 같은 데이터를 다룰 때가 이 작업을 서로 분리하는 단위가 된다

- 트랜잭션은 전체가 수행되거나 또는 전혀 수행되지 않아야 한다 (All or Nothing)
  <br/> <br/>
  
- 예를 들면 계좌 이체를 할 때, A 계좌에서 B 계좌로 돈이 입금된다고 하면 2개의 UPDATE 문이 나간다 <BR/>
  이 두가지 UPDATE문이 하나의 트랜잭션으로 묶이게 된다

### 데이터베이스 시스템에서 동시성 제어
- 다중 사용자 환경에서 둘 이상의 트랜잭션이 동시에 수행될 때, 일관성을 해치지 않도록 트랜잭션의 데이터 접근 제어
  
- 한번에 다수의 DB 접근을 필요로 하는 요청이 들어올 때 트랜잭션 단위로 DB 접근을 하게 되는데<BR/>
  DB 접근을 동시적으로 모두 허용해주면 데이터베이스의 일관성과 무결성이 깨지게 됌

### 동시성 제어 기법
기법 이름	                          장점	                                                        단점

Locking 기법	        1. 데이터 오류 가능성 사전에 예방                            1. Lock 대기 시간 -> 동시성 저하
                      2. 구현하기 쉬움	                                          2. 교착 상태(DeadLock) 발생 가능성 존재
                    
TimeStamp	            1. 교착 상태(DeadLock) 발생 가능성 없음                      1. Rollback 발생 확률이 높음
                      2.트랜잭션 대기 시간 없음	
                      
적합성 검증(낙관적)	  1. 동시 처리 능력 증가                                       1. 장기 트랜잭션 철회시 자원 낭비
                      2. 트랜잭션 대기 시간 없음	
                      
MVCC	                1. 트랜잭션 대기 시간 없음                                   1. 사용하지 않는 데이터를 정리하는 시스템이 필요
                      2. 다른 트랜잭션이 해당 데이터를 수정해도 영향받지 않음       2. 데이터 버전의 충돌이 일어날 수 있음
                                                                                  3. UNDO 블록, I/O 등의 부가적인 오버헤드 발생
