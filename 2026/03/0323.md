# 3/23 Redis, 캐싱, 기타 공부

### Redis 사용 의미

- 인메모리 데이터베이스이며, 지연이 적은 읽기 / 쓰기 성능을 가짐
- 일시적인 데이터, 변경이 잦은 데이터를 다뤄야 되는 상황에서 많이 활용됨

### Redis 실제 사용 예시

- @RedisHash("item") 를 통해 이 데이터(클래스)가 Redis에 저장될 것임을 의미
- 그 후 RedisTemplate<String, ItemDto> 이런 식으로 레디스 템플릿 사용 가능

### 캐싱

- Cache는 본래 CPU 내부의 작은 영역으로, 빈번히 접근하게 되는 데이터를 저장해두는 임시 기억 장치
- 웹 개발자가 직접적으로 다루는 부분은 아니지만, 캐시의 목적과 방식을 웹 개발에 적용해,
  빈번하게 접근하게 되는 데이터베이스의 데이터를 Redis 등의 인메모리 데이터베이스에 저장을 함으로서
  데이터를 조회하는데 걸리는 시간과 자원을 감소시키는 기술을 캐싱이라고 함
- 캐싱을 하려면 관련 어노테이션 사과 Bean 등록 등 필요한 것들을 해 주어야 함. 아래와 같은 예시
```
@Cacheable(cacheNames = "itemCache", key = "args[0]")
```
